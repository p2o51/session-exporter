# 浏览与导出

## 浏览与筛选

来自全部四种工具的每一条会话都会出现在同一份列表中。左侧栏决定显示哪些内容：

- **搜索** —— 匹配标题、项目和模型。
- **来源** —— 开关 Claude Code / Codex / Cursor / Antigravity（每个标签都显示其会话数）。
- **项目** —— 会话所属的文件夹。该列表可搜索，并按会话数排序。
- **日期范围** —— 针对会话更新日期的起止筛选。

可按**最近更新**、**最近创建**、**成本最高**、**token 最多**、**消息最多**或**标题 A–Z** 对表格排序。

右上角的计数器显示你的会话总数中有多少条符合当前筛选，连同它们合计的 token 和成本。

## 阅读一条会话

点击某一行可打开详情抽屉。它会显示：

- 标题、来源、项目路径、模型、消息数和日期范围。
- 一个 **token 网格**：合计 / 输入 / 输出，外加缓存读取、缓存写入和推理（在来源有记录时），以及**缓存命中率**。
- 该会话的**估算成本**。
- 完整的**会话记录**，每一轮按角色（用户、助手、推理、系统）用颜色区分并带时间戳。极长的记录只渲染前 500 条消息；导出即可获得完整文本。

## 选择

- 勾选某一行的复选框即可选中它。选择会**跨筛选保持** —— 缩小列表、选中更多、再放宽，你之前的选择仍然保留。
- **全选**会选中*当前筛选*下的每一条会话。取消勾选（或使用 **Clear**）即可重置。

只要有任何内容被选中，底部就会出现一个栏，显示数量、合计 token、成本和缓存命中率，并带有两个导出按钮。

## 导出 ZIP

**Export ZIP** 会为所选会话生成一个自包含的归档：

```
sessions-export.zip
├── index.json          # 每条会话的元数据，外加你使用的筛选条件
├── index.csv           # 相同的元数据，以扁平表格形式呈现
├── README.txt
└── sessions/
    ├── <source>__<id>.json   # 完整元数据 + 会话记录（机器可读）
    └── <source>__<id>.md     # 相同的会话记录，人类可读
```

`index.json` 和 `index.csv` 承载了所有元数据列 —— 来源、项目、日期、模型、消息数、每一项 token 计数、缓存命中率，以及**成本** —— 因此这份导出是你所选内容的完整、可移植的记录。

## 导出到 Notion

**Export for Notion** 会生成一个形态不同、专为 Notion 导入器设计的归档 —— 完整的操作步骤参见 [导出到 Notion](/zh/guide/notion)。

## 重新扫描

索引会被缓存以提升速度。如果自启动以来你有了新会话，点击顶栏的 **Refresh** 即可重新扫描你的会话文件。（只是编辑了价格？那就改用 Stats 面板里的 **↻ Reload prices** —— 它是瞬时的，且不会重新扫描。）
